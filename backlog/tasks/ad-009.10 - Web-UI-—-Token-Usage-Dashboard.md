## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Display token usage/quota statistics per agent in the Web UI. Data sources per agent:

**Codex:** Read usage via Codex CLI app-server JSON-RPC or parse local session data. Reference: steipete/CodexBar (macOS menu bar app, multi-provider, uses codex app-server for JSON-RPC data + OpenAI web dashboard cookies for limits). Also see Finesssee/Win-CodexBar for Windows port.

**Claude Code:** Parse local session logs at ~/.claude/projects/**/*.jsonl (token counts per message) and ~/.claude/stats-cache.json (pre-aggregated daily stats, model usage). Reference: claude-code-stats (HTML dashboard from local data), masorange/ClaudeUsageTracker (macOS real-time tracker).

**Cursor:** Admin/Analytics API (team plans) or parse local VS Code extension storage. Reference: Ittipong/cursor-price-tracking (VS Code extension), lixwen/cursor-usage-monitor, ofershap/cursor-usage-tracker (Enterprise dashboard).

**Gemini CLI:** Check if local usage logs exist in ~/.gemini/ or parse session output.

**Other agents:** Best-effort — check for local log files, session data, or API endpoints.

Implementation: src/usage/ module with per-agent usage readers. API: GET /api/usage, GET /api/usage/:agentId. Frontend: Usage cards/charts showing tokens used, estimated cost, quota remaining, reset time. Start with Codex + Claude Code as MVP, expand to others.
